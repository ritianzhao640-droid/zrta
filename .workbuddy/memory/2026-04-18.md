- 输入框空 → 从 `distributor.burnInfo(user).inviter` 读链上已绑定上级
- 都没有 → 尝试 sessionStorage 存储值

## 20:51 — 累计销毁显示偏差（烧10万显示17万）

### 根因分析
**双重除法 bug**：
1. `loadUserBurnDistributorStats()` 中 `chainData.userBurned = burnInfo.rawBurned`（存的是原始 BigNumber）
2. 但 Fix 1 修改后错误地覆盖为已转换数值：`chainData.userBurned = burnedNum`
3. 邀请页进度条又执行了 `formatUnits(chainData.userBurned, 18)` **再次除以 10^18**
4. 结果：实际值被除以了两次，或因数据类型不一致导致转换异常

### 修复方案
1. **累计销毁显示（#myCumulativeBurn）**
   - 改用 `formatUnits(rawVal.toString(), 18)` 显式指定 18 位
   - 加调试日志：打印原始值和转换后值
   - 安全校验：>10亿 或非有限数时回退 '--'
   - 新增 `chainData.userBurnedDisplay` 存储已转换值

2. **邀请页进度条 myBurned**
   - 优先读取 `chainData.userBurnedDisplay`（已转换，不二次除法）
   - 兜底：从原始 `chainData.userBurned` 转换
   - 再兜底：Lens 的 `inviterBurnedAmount`
   - 加日志标注当前使用的数据源

3. **销毁权重计算**
   - `burnInfo.weight` 和 `chainData.totalBurned` 统一用 `formatUnits(xxx, 18)`
   - 避免 formatEther 内部处理 BigNumber 不一致的问题
