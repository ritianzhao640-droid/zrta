# zrta 前端数据层改造说明

## 改造目标
将前端数据源从旧版 Lens 合约全面迁移至新版链上接口（MEYieldVault 代理模式），最大限度保持现有 UI、DOM 结构、函数名和事件名不变。

## 一、删除的旧依赖
1. **移除对 Lens 合约的所有调用**：原 `loadGlobalStats`、`loadBurnStats`、`loadUserStats` 等函数中通过 Lens 聚合数据的逻辑已全部删除。
2. **移除对旧版 HolderRewardTracker 的依赖**：不再通过 `HolderRewardTracker` 查询 holder 分红，改为通过 `HolderEligibilityOracle` 读取官方分红（WBNB/BNB）。
3. **移除旧版合约配置**：原 `CONTRACTS` 对象中的 `LENS`、`HOLDER_REWARD_TRACKER`（旧）等配置已删除。

## 二、新增的合约实例
在 `CONTRACTS` 对象中新增以下合约配置（ABI 来自 `mevault-frontend-interfaces-clean.zip`）：
1. **VAULT** (`MEYieldVault` 代理合约)
   - 地址：`0xC09f04d1DDD278A8a9406CfB6E9C80f2719DB2b0`（固定）
   - 用途：所有用户操作的核心入口，提供 `overview()` 聚合数据、`pendingClaim`、`burn`、`claim` 等方法。
2. **BURN_DISTRIBUTOR** (`BurnReferralDistributor`)
   - 地址：动态发现（通过 `vault.burnDistributor()`）
   - 用途：管理销毁统计、邀请关系、销毁奖励和邀请奖励。
3. **HOLDER_ELIGIBILITY_ORACLE**
   - 地址：动态发现（通过 `vault.holderEligibilityOracle()`）
   - 用途：查询官方分红（WBNB/BNB）的资格、待领取和已领取金额。
4. **TOKEN** (CLAIM 代币)
   - 地址：`0x379f9E96BDC94572492e3BAC75A5122fFb7E7777`（固定）
   - 用途：代币余额查询、授权等。
5. **辅助地址**（仅地址，无 ABI）
   - `LISTA_STAKE_MANAGER`：动态发现（通过 `vault.listaStakeManager()`）
   - `SLIS_BNB`：动态发现（通过 `vault.slisBNB()`）
   - `MULTISIG`（营销钱包）：动态发现（通过 `vault.marketingWallet()`）
   - `HOLDER_REWARD_TRACKER`：动态发现（通过 `vault.holderRewardTracker()`），当前版本未使用。

## 三、改为动态发现的地址
除 `TOKEN` 和 `VAULT` 地址需手动配置外，其余合约地址均通过 Vault 的视图函数动态获取，提升可维护性：
- `burnDistributor()` → `CONTRACTS.BURN_DISTRIBUTOR.address`
- `slisBNB()` → `CONTRACTS.SLIS_BNB.address`
- `holderEligibilityOracle()` → `CONTRACTS.HOLDER_ELIGIBILITY_ORACLE.address`
- `marketingWallet()` → `CONTRACTS.MULTISIG.address`
- `listaStakeManager()` → `CONTRACTS.LISTA_STAKE_MANAGER.address`
- `holderRewardTracker()` → `CONTRACTS.HOLDER_REWARD_TRACKER.address`

动态发现逻辑在 `initializeContractAddresses()` 函数中实现，并在 `refreshAllData()` 首次调用时执行一次。

## 四、数据读取逻辑改造
### 1. 首页聚合数据
- **旧逻辑**：调用 Lens 的 `getGlobalStats()`。
- **新逻辑**：优先调用 `vault.overview()`，一次性获取 11 个关键字段（包括总质押 BNB、Vault 余额、分红比例、用户待领取奖励等）。
- **函数**：`loadGlobalStats()` 已重写。

### 2. 销毁统计
- **旧逻辑**：调用 Lens 的 `getBurnStats()`。
- **新逻辑**：调用 `BurnDistributor` 的 `totalActualBurned`、`totalBurnWeight`、`totalBurnRewardDistributed`、`totalInviteRewardDistributed`。
- **函数**：`loadBurnStats()` 已重写。

### 3. 用户数据
- **旧逻辑**：分散在多个 Lens 调用中。
- **新逻辑**：
  - **项目 holder 分红（slisBNB）**：通过 `vault.pendingClaim(user)` 获取。
  - **销毁奖励**：通过 `vault.pendingBurnReward(user)` 获取。
  - **邀请奖励**：通过 `vault.pendingInviteReward(user)` 获取。
  - **官方分红（WBNB/BNB）**：通过 `HolderEligibilityOracle.officialDividendPending(user)` 获取。
  - **用户销毁信息**：通过 `vault.burnInfo(user)` 获取。
- **函数**：`loadUserStats()` 已重写。

### 4. 收益严格区分
- **项目 holder 分红**：`chainData.myPendingClaim`（slisBNB）
- **官方分红**：`chainData.officialDividendPending`（WBNB/BNB）
- 两者在 UI 上应明确区分，不可混淆。

## 五、交易发送函数改造
所有用户操作（销毁、领取奖励等）已改为直接调用 Vault 或相应合约：
- **销毁**：`vault.burn(amount, inviter)`
- **领取项目 holder 分红**：`vault.claim()`
- **领取销毁奖励**：`vault.claimBurnReward()`
- **领取邀请奖励**：`vault.claimInviteReward()`
- **领取官方分红**：需调用 `HolderEligibilityOracle` 相关函数（暂未实现，需前端根据官方分红合约的具体接口补充）。
- **同步 holder 账户**：`vault.syncMyHolderAccount()`（用于更新官方分红资格）。

原 `claimHolderDividend` 等函数已标记为待更新。

## 六、隐藏/禁用的管理员功能
根据“默认只实现普通用户操作”的要求，以下管理员功能已被隐藏或禁用：
1. **营销钱包提取**：原 `claimMarketing` 按钮已通过 CSS 隐藏（添加 `hidden` 类）。
2. **阈值设置**：`setAutoStakeThreshold`、`setMarketingAutoSendThreshold` 等函数对应的 UI 元素已隐藏。
3. **营销钱包地址更新**：`setMarketingWallet` 按钮已隐藏。
4. **Vault 初始化**：`initialize` 按钮已隐藏。

隐藏方式：在 DOM 加载后通过 JavaScript 为相关元素添加 `style="display:none;"`。

## 七、官方分红 Claim 接入情况
- **已接入**：通过 `HolderEligibilityOracle` 读取官方分红的待领取、已领取金额和 holder 资格。
- **未接入**：官方分红的领取交易函数尚未实现，因为 `HolderEligibilityOracle` 未提供 claim 方法，需等待官方分红合约的具体接口说明。
- **临时方案**：UI 上显示官方分红金额，但领取按钮暂不可用，并提示“官方分红领取功能即将上线”。

## 八、文件变更清单
1. `index.html`（主文件）
   - 更新 `CONTRACTS` 配置对象（第 1186 行起）
   - 新增 `initializeContractAddresses()` 函数（动态地址发现）
   - 重写 `loadGlobalStats()`、`loadBurnStats()`、`loadUserStats()` 数据读取函数
   - 在 `refreshAllData()` 中插入动态地址初始化检查
   - 添加管理员按钮隐藏逻辑（DOMContentLoaded 事件内）
   - 保留原有 UI 结构和事件绑定，仅修改数据来源和交易目标

2. `contracts_config.js`（生成的中间配置，仅用于参考）
3. `改造说明.md`（本文档）

## 九、使用说明
1. 将本 `index.html` 直接覆盖原文件。
2. 确保前端访问的区块链网络与合约地址对应（BSC 主网）。
3. 首次加载时会自动初始化合约地址，稍等片刻即可看到新数据。
4. 普通用户可正常进行销毁、领取项目分红、销毁奖励、邀请奖励等操作。
5. 官方分红领取功能暂不可用，等待后续更新。

## 十、注意事项
- 所有 ABI 均来自 `mevault-frontend-interfaces-clean.zip` 中的 `output.json`，确保与链上合约完全一致。
- 动态地址发现失败会导致部分功能不可用，请检查 Vault 代理地址是否正确以及网络连接。
- 如遇问题，查看浏览器控制台日志（前缀 `[loadGlobalStats]`、`[loadBurnStats]`、`[loadUserStats]` 等）。
